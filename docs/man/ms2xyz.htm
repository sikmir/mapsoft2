NAME

ms2xyz -- writing track in a text form

USAGE

        ms2view <options> <input files>

GENERAL OPTIONS:

--tshift <arg> -- time shift, hours

-t, --tfmt <arg> -- time format string for %T field (default: "%F %T")

-f, --fmt <arg> -- format string (default: "%x %y %z %T %D %S")

-w, --win <arg> -- window for speed calculation, sec (default: 120)

-b, --break <arg> -- place to break calculation and put empty line (none | day | track, default: none)

-h, --help -- show help message

--pod -- show help message as POD template

-v, --verbose -- be verbose

-o, --out <arg> -- output file

If output file is not specified the data is printed to stdout.

READING GEODATA

See GEODATA FORMATS section in mapsoft2(5) for supported formats and related options.

FORMAT

        %% -- % sign
        %x -- wgs lon
        %y -- wgs lat
        %z -- altitude, m
        %t -- time (seconds since 1970-01-01 00:00:00 UTC)
        %T -- date and time in ISO 8601 form (<yyyy-mm-dd>T<HH:MM:SS.FFF>Z)
        %u -- time from the previous point, s
        %d -- % distance from part beginning, km
        %D -- % distance from data beginning, km
        %e -- % distance from the previous point, m
        %S -- % speed, km/h
        %n -- %  point number from part beginning
        %N -- %  point number from data beginning
        %p -- %  part number

EXAMPLES

Print track as a 3-column table with unix time (seconds), distance from the beginning, and altitude:

        $ ms2xyz mytrack.plt -f "%t %D %z"
        1559876972 0.000 1238.8
        1559876995 0.031 1226.3
        1559877026 0.050 1185.4
        1559877060 0.051 1186.9
        ...
